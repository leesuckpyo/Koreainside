# Korea Inside Project Memory

## 목적

프로젝트에서 이미 확정된 사항은
다시 토론하거나 변경하지 않는다.

변경이 필요한 경우
변경 사유와 승인 기록을 남긴다.

Project Memory는 과거 결정과 승인 내용을 확인하는 참고 기준이다.

파일 생성, 수정, 삭제, 이동, 이름 변경, 리팩토링, 포맷팅, stage, commit, push 권한을 자동 승인하지 않는다.

실제 작업 권한은 현재 사용자 명시 지시, 승인된 작업 범위, root `AGENTS.md`, Conflict Resolution Standard를 우선한다.

과거 memory는 현재 작업 범위를 자동 확장하지 않는다.

변경이 필요해 보이면 먼저 문제점, 영향 범위, 대상 파일, 대안, 권장안을 보고하고 승인 대기한다.

---

## Memory Level

### Level A (Locked)

절대 변경하지 않는 결정

예)

- 프로젝트명

- 도메인

- 브랜드

- 디자인 철학

- AI 원칙

- SEO 철학

변경 시 반드시 승인 필요.

---

### Level B (Approved)

확정되었지만
향후 개선 가능한 사항

예)

페이지 구조

컴포넌트

FAQ

이미지 정책

---

### Level C (Draft)

실험 단계

사용자 명시 승인 범위 안에서만 조정 가능한 낮은 위험도 항목.

## AI 규칙

AI는

이미 확정된 사항을

다시 제안하지 않는다.

동일한 논의를 반복하지 않는다.

기존 결정을 우선 참조한다.

단,

기존 결정은 사용자 명시 지시나 root `AGENTS.md`보다 우선하지 않는다.
