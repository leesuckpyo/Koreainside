# Korea Inside Decisions

This document is a historical project decision memo / summary.

The official ADR / Decision Log is `../decision-log.md`.

If this document conflicts with `../decision-log.md`, follow `../decision-log.md`.

## Decision 001 - Problem-solving content first
Official ADR: `../decision-log.md` ADR-011 Problem-solving Content First.

Summary:
Korea Inside focuses on real travel problems foreigners face in Korea, not general tourist attraction introductions.

Reason:
Generic tourist content is easy to copy and hard to differentiate.

## Decision 002 - HTML text first
Official ADR: `../decision-log.md` ADR-002 HTML First.

Summary:
Essential information must be written as HTML text, not embedded only in images.

## Decision 003 - No mobile app first
Official ADR: `../decision-log.md` ADR-012 No Mobile App First.

Summary:
Development order is Landing Page -> Website -> PWA -> Mobile App.

Reason:
Traffic and demand must be validated before app development.

## Decision 004 - Remove duplicate content
Related official ADR: `../decision-log.md` ADR-004 Less is More.

Summary:
If an infographic already explains the concept, repeated cards or tables should be removed unless they add new decision-making value.

Reason:
Better readability, shorter pages, and clearer UX.
