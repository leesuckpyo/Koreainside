# Korea Inside AI Self Audit Standard

------------------------------------------------

작업 시작 전

------------------------------------------------

□ Product Vision을 확인했는가?

□ Project Memory를 확인했는가?

□ 이미 확정된 결정을 다시 변경하려고 하지 않는가?

□ 기존 기능으로 해결 가능한가?

□ 새로운 기능이 정말 필요한가?

□ 기존 컴포넌트를 재사용할 수 있는가?

□ 이번 변경 범위를 이해했는가?

------------------------------------------------

구현 중

------------------------------------------------

□ 요구사항을 벗어나지 않았는가?

□ 기존 기능을 삭제하지 않았는가?

□ SEO를 해치지 않았는가?

□ HTML Text를 유지했는가?

□ 불필요한 리팩토링을 하지 않았는가?

□ 공통 컴포넌트를 우선 사용했는가?

------------------------------------------------

구현 완료 후

------------------------------------------------

□ 요구사항을 모두 만족하는가?

□ 관련 md 문서를 업데이트했는가?

□ Review Checklist를 통과하는가?

□ Code Review Standard를 통과하는가?

□ Definition of Done을 만족하는가?

------------------------------------------------

금지 사항

------------------------------------------------

□ 승인 없는 구조 변경

□ 추측 기반 구현

□ 과도한 개선

□ 사용자가 요청하지 않은 기능 추가

□ 이미 결정된 사항 재논의

------------------------------------------------

최종 질문

------------------------------------------------

이번 작업은

사용자의 요청을

정확히 해결했는가?

아니면

AI가 하고 싶은 작업을 수행했는가?

------------------------------------------------

최종 원칙

------------------------------------------------

AI는

많이 수정하는 것이 아니라

정확하게 수정하는 것이 목표이다.
