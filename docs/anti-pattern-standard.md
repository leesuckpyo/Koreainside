# Korea Inside Anti-Pattern Standard

================================================

목적

================================================

좋은 프로젝트는

무엇을 할지뿐 아니라

무엇을 하지 않을지도

명확해야 한다.

================================================

Product

================================================

하지 않는다.

□ 관광지 나열

□ 페이지 수 늘리기

□ 조회수용 콘텐츠

□ 광고 중심 설계

□ 기능만 많은 사이트

================================================

UX

================================================

하지 않는다.

□ 긴 설명

□ VS 남발

□ 클릭 유도 UX

□ 팝업 남발

□ 자동 재생 영상

□ 과도한 애니메이션

================================================

Content

================================================

하지 않는다.

□ 추측 작성

□ 과장 표현

□ 확인되지 않은 정보

□ 오래된 정보 방치

□ 불필요한 중복

================================================

SEO

================================================

하지 않는다.

□ 키워드 남발

□ AI 생성 저품질 콘텐츠

□ 의미 없는 랜딩페이지

□ 중복 메타

□ 클릭베이트 제목

================================================

Development

================================================

하지 않는다.

□ 승인 없는 구조 변경

□ 중복 코드

□ 중복 컴포넌트

□ 중복 CSS

□ 사용하지 않는 JS

================================================

AI

================================================

하지 않는다.

□ 이미 결정된 내용 재제안

□ 요청하지 않은 기능 추가

□ 불필요한 리팩토링

□ Project Memory 무시

□ Diff 없이 수정

================================================

Business

================================================

하지 않는다.

□ 신뢰보다 수익 우선

□ 제휴를 먼저 노출

□ 광고로 UX 훼손

□ 사용자를 특정 서비스로 유도

================================================

Final Principle

================================================

Korea Inside는

좋은 기능을 많이 만드는 프로젝트가 아니라

나쁜 선택을 하지 않는 프로젝트이다.
