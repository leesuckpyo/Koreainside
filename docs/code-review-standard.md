# Korea Inside Code Review Standard

## 목적

모든 코드 변경은
작동 여부뿐 아니라
품질, 유지보수성, UX, SEO까지 함께 검토한다.

---

## HTML Review

□ Semantic HTML 사용

□ Heading 구조 확인

□ HTML Text 유지

□ Accessibility 확인

□ 불필요한 div 제거

---

## CSS Review

□ 기존 Design System 준수

□ 중복 CSS 없음

□ 반응형 확인

□ 공통 클래스 재사용

□ 불필요한 스타일 제거

---

## JavaScript Review

□ 기존 코드 재사용

□ 전역 변수 최소화

□ 이벤트 중복 없음

□ 오류 처리 포함

□ 성능 영향 확인

---

## UX Review

□ Hero 명확한가

□ CTA 적절한가

□ 사용자가

"아~ 이거구나."

를 느끼는가

□ Title

□ Meta

□ WebP 확인
