항상 같은 우선순위로 판단한다.

------------------------------------------------

대표 충돌 사례

------------------------------------------------

SEO vs UX

→ 사용자 이해를 우선하면서 SEO 손실을 최소화한다.

Design vs Accuracy

→ 정확성을 우선한다.

AI 제안 vs 확정 결정

→ 확정 결정을 따른다.

새 기능 vs 기존 구조

→ 기존 구조 재사용을 우선 검토한다.

속도 vs 품질

→ 품질을 우선한다.

실물 사진 vs AI 이미지

→ 실물 정확성이 중요한 경우 사용자 제공 실물 사진을 우선한다.

------------------------------------------------

판단 절차

------------------------------------------------

1. 충돌 내용 정의

2. 영향 범위 분석

3. 우선순위 적용

4. 대안 비교

5. 권장안 제시

6. 사용자 승인

7. 구현

------------------------------------------------

예외 규칙

------------------------------------------------

긴급 오류 수정은
간소화 절차를 적용할 수 있다.

단,
사후 변경 기록을 남긴다.

------------------------------------------------

최종 원칙

------------------------------------------------

충돌은

AI가 임의로 해결하지 않는다.

우선순위를 적용하고

사용자의 승인을 받는다.
