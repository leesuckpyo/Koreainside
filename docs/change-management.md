# Korea Inside Change Management Standard

## 목적

프로젝트의 모든 중요한 변경 사항을 기록하여
추후 유지보수와 회귀(Regression) 방지에 활용한다.

---

## 변경 대상

다음 항목은 반드시 기록한다.

- 페이지 구조 변경
- Hero 변경
- SEO 변경
- FAQ 변경
- 이미지 정책 변경
- 디자인 시스템 변경
- 컴포넌트 변경
- 공통 CSS 변경
- 공통 JS 변경
- 내부 링크 구조 변경

## 변경 절차

1. 변경 필요성 설명
2. 영향 범위 분석
3. Diff 제시
4. 사용자 승인
5. 수정
6. 문서 업데이트
7. 결과 보고

---

## 긴급 수정

오탈자, 깨진 링크, 단순 버그는
간소화 절차를 적용할 수 있다.

단, 수정 내역은 기록한다.

---

## 금지 사항

- 승인 없이 구조 변경
- 기록 없는 공통 파일 수정
- 이유 없는 SEO 변경
- 문서와 코드 불일치

---

## 최종 원칙

모든 중요한 변경은
'왜 변경했는지'를
누구나 이해할 수 있어야 한다.
