# Korea Inside Change Management Standard

## 목적

프로젝트의 모든 중요한 변경 사항을 기록하여
추후 유지보수와 회귀(Regression) 방지에 활용한다.

Change Management는 변경 판단과 기록 필요성 검토 기준이다.

파일 생성, 수정, 삭제, 이동, 이름 변경, 리팩토링, 포맷팅, stage, commit, push 권한을 자동 승인하지 않는다.

실제 작업 권한은 현재 사용자 명시 지시, 승인된 작업 범위, root `AGENTS.md`, Conflict Resolution Standard를 우선한다.

변경 기록, 관련 MD, Change Log, Project Memory, Decision Log 업데이트는 사용자가 명시 요청하거나 승인한 범위 안에서만 수행한다.

기록이 필요해 보이면 먼저 대상 문서, 이유, 영향 범위, 대안, 권장안을 보고하고 승인 대기한다.

## 변경 절차

1. 변경 필요성 설명
2. 영향 범위 분석
3. Diff 제시
4. 사용자 승인
5. 수정
6. 승인된 범위 안에서 필요한 변경 기록 또는 문서 업데이트
7. 결과 보고

---

## 긴급 수정

오탈자, 깨진 링크, 단순 버그는
간소화 절차를 적용할 수 있다.

단, 수정 내역 기록이 필요하면 대상 문서와 이유를 보고하고 승인 후 수행한다.

---

## 금지 사항

- 승인 없이 구조 변경
- 사용자 승인 범위 밖 공통 파일 수정
- 이유 없는 SEO 변경
- 문서와 코드 불일치
- 사용자 명시 승인 없는 변경 기록 또는 문서 업데이트
