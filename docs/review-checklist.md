# Korea Inside Review Checklist

## 1. SEO

□ H1은 페이지당 1개인가?

□ Meta Title 작성

□ Meta Description 작성

□ Canonical 확인

□ Open Graph 작성

□ Structured Data 확인

□ 내부 링크 확인

□ URL 구조 확인

---

## 2. HTML

□ 중요한 내용이 HTML Text인가?

□ 이미지 안에 핵심 정보를 넣지 않았는가?

□ 브라우저 자동번역이 가능한가?

□ Heading 구조(H1~H3)가 올바른가?

---

## 3. UX

□ Hero에서 페이지 목적이 즉시 이해되는가?

□ 사용자가 "아~ 이거구나."를 느낄 수 있는가?

□ CTA가 자연스러운가?

□ 긴 문단이 없는가?

□ 모바일에서 읽기 쉬운가?

---

## 4. 이미지

□ 실물 사진이 필요한 부분은 확보했는가?

□ AI 이미지가 적절한가?

□ 이미지가 정보를 전달하는가?

□ 단순 장식 이미지가 아닌가?

□ alt 속성이 작성되었는가?

□ WebP 사용 여부 확인

---

## 5. 애니메이션

□ 필요한 곳에만 사용했는가?

□ 튜토리얼이 아니라 이해를 돕는가?

□ 과도한 움직임이 없는가?

---

## 6. 콘텐츠

□ 정보가 최신인가?

□ 공식 자료 기준인가?

□ 과장 표현이 없는가?

□ 객관적인 설명인가?

□ 비교를 남발하지 않았는가?

---

## 7. 접근성

□ Contrast 확인

□ Keyboard Navigation 확인

□ Focus 확인

□ aria-label 확인

---

## 8. 성능

□ Lazy Loading

□ 이미지 최적화

□ Font 최적화

□ CLS 문제 없음

□ LCP 문제 없음

---

## 9. 링크

□ 내부 링크 확인

□ 외부 링크 확인

□ Broken Link 없음

---

## 10. 제휴

□ 정보 제공 후 제휴를 배치했는가?

□ 제휴가 과도하지 않은가?

□ 사용자가 스스로 선택할 수 있는가?

---

## 11. 최종 공개

□ Desktop 확인

□ Tablet 확인

□ Mobile 확인

□ 오타 검사

□ 맞춤법 검사

□ 디자인 일관성 확인

---
