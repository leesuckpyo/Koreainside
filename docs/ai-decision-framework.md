# Korea Inside AI Decision Framework

## 우선 기준

이 문서는 AI 판단 프레임워크이다.

파일 변경 권한을 자동 승인하지 않는다.

현재 사용자 명시 지시, 승인된 작업 범위, root `AGENTS.md`, Conflict Resolution Standard를 우선한다.

## AI는 먼저 구현하지 않는다.

반드시 아래 순서대로 판단한다.

-------------------------------------------------

STEP 1

문제 정의

-------------------------------------------------

사용자가 해결하려는 문제는 무엇인가?

기존 페이지에서 해결 가능한가?

새 기능이 필요한가?

-------------------------------------------------

STEP 2

Product Vision 확인

-------------------------------------------------

Product Vision과 일치하는가?

-------------------------------------------------

STEP 3

Project Memory 확인

-------------------------------------------------

이미 확정된 결정과 충돌하는가?

기존 방향을 변경하는가?

-------------------------------------------------

STEP 4

Decision Rules 확인

-------------------------------------------------

정말 필요한 기능인가?

사용자 가치가 있는가?

-------------------------------------------------

STEP 5

Design System 확인

-------------------------------------------------

기존 컴포넌트 재사용 가능한가?

새 디자인이 필요한가?

-------------------------------------------------

STEP 6

SEO 확인

-------------------------------------------------

검색 가치가 있는가?

새 URL이 필요한가?

-------------------------------------------------

STEP 7

Implementation 판단

-------------------------------------------------

아래 분류는 판단용이며 실행 승인이 아니다.

수정

추가

삭제

리팩토링

중 어느 것인가?

-------------------------------------------------

STEP 8

영향 분석

-------------------------------------------------

SEO 영향

UX 영향

Performance 영향

Accessibility 영향

-------------------------------------------------

STEP 9

Diff 작성

-------------------------------------------------

무엇이 변경되는가?

영향 범위는?

-------------------------------------------------

STEP 10

승인 대기

-------------------------------------------------

승인 전에는 수정하지 않는다.

-------------------------------------------------

STEP 11

수정

-------------------------------------------------

승인 후 수정

-------------------------------------------------

STEP 12

문서 업데이트

-------------------------------------------------

문서 업데이트는

사용자가 명시 요청하거나 승인한 범위 안에서만 수행한다.

관련 md,

Change Log,

Project Memory,

Decision Log 업데이트도

자동 절차가 아니다.

필요해 보이면

대상 문서,

이유,

영향 범위,

권장안을 보고하고

승인 대기한다.

-------------------------------------------------

최종 원칙

AI는

'빠르게 만드는 것'

보다

'올바르게 판단하는 것'

을 우선한다.

AI 판단은 사용자 명시 승인보다 우선하지 않는다.
